# 在庫の管理

Stockは、販売チャネル（web サイト）のソースに対する製品のバーチャルな集計在庫を表します。 サイトの設定によっては、在庫が1つ以上の販売チャネルに割り当てられる場合があります。 各販売チャネルには1つの在庫のみを割り当てることができ、1つの在庫を複数の販売チャネルに割り当てることができます。 在庫を通じて、販売チャネルを通じて注文が発生した際に使用するソースの優先順位付けを変更できます。

削除または無効化できないデフォルトのストックから開始します。 追加の販売チャネルは、在庫にのみ追加できます。 割り当てられたソースはデフォルトのSourceのみです。 この在庫は、シングルソースの販売者、サードパーティの統合、および輸入製品によって使用されます。

販売チャネル：在庫を販売するエンティティを表します。 既定では、[!DNL Commerce]はストア web サイトを販売チャネルとして提供します。 セールスチャネルを拡張して、B2B顧客グループやストアビューなどの追加チャネルをサポートすることも可能です。 各販売チャネルは、1つのStockにのみ関連付けることができます。

- **Sales Channel サポート** – 現在、販売チャネルには標準のWeb サイトが含まれています。 B2B顧客グループやストアビューなどのカスタムオプションを含めるように販売チャネルを拡張することができます。 各販売チャネルには、1つの在庫のみを割り当てることができます。 1つの在庫を複数の販売チャネルに割り当てることができます。
- **ソースにマッピング** – 各在庫には、1つ以上の有効または無効なソースを割り当てることができ、製品ごとにバーチャル集計在庫を計算します。
- **優先順位付けフルフィルメント** - Source Selection Algorithmのすぐに使用できる優先順位付けアルゴリズムは、注文を処理する際に、在庫のソースリストを上から下に使用します。

## マウンテンバイクとマウンテンストアの在庫例

すべてのストアは、デフォルトのストックから開始されます。 次の理由により、`Enabled`のままである必要があります。

- 新しい製品を読み込むときに使用され、デフォルトのソースと在庫に製品が自動的に割り当てられ、[!DNL Inventory Management]に即座にアクセスできるようになります。
- このストックにデフォルトのSource以外のソースを追加することはできません。
- これは、シングルソースマーチャント、バンドル製品、およびグループ化された製品で必要であり、使用されます。

マルチソースのマーチャントの場合は、店舗と注文のフルフィルメントに最適なストックを作成し、設定します。 販売チャネルに新規在庫を割り当てると、その販売チャネル内の既存の在庫はすべて未割り当てになります。

マルチストアのインストールの場合、初期設定のStockは、最初に[ メイン Web サイト ](../stores-purchase/stores.md#add-websites){target="_blank"}と初期設定のストアに割り当てられます。 **[!UICONTROL Products]** グリッドビューには、有効な商品と無効な商品に対して、正しい在庫と数量が表示されます。

| 列 | 説明 |
|--|--|
