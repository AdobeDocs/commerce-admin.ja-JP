# 注文ステータスと予約

[!DNL Inventory Management] は、注文ごとの部分的および完全な請求、支払い、発送、およびキャンセルをサポートします。 処理、請求、出荷、潜在的な払い戻しを通じて注文を管理する場合、 [!DNL Commerce] 予約を自動的に入力または変更して、在庫（または販売チャネル）の販売可能数量と、ソースごとの手持在庫数量を更新します。 積極的にアクセスしたり、予約を入力したりする必要はありません。 処理を完了し、キャンセル、または払い戻し注文を行う場合は、それを実行します。

これらの予約は、常に販売可能数量を調整し、正または負の数量を使用して数量を増減します。 その結果、最新の製品可用性に対する手持在庫と販売可能数量が更新されます。

オーダーと出荷について詳しくは、 [オーダーと出荷の管理](shipments.md).

## オーダー管理オプション

在庫ステータスと顧客の要求に応じて、一部の支払いと取り消し、複数のソースからの一部出荷、逆注文、または返品された製品を返金するクレジット・メモを使用してオーダーを更新できます。

### 出荷

請求後、注文全体を満たすまで、一部または全部の出荷を送付します。 各出荷は予約を換算し、ソースごとの製品数量から金額を差し引きます。 在庫の販売可能数量を更新するには、予約報酬を入力します。 一部出荷を送付する場合、各出荷は製品数量と予約からその金額を差し引きます。 出荷前の製品予約は、出荷前の状態のまま残るので、販売可能な数量が最新になり、製品在庫の管理や複数のソース出荷とバックオーダーのサポートが可能になります。

### キャンセルされた注文

顧客が出荷前（一部または全部）に注文をキャンセルした場合、在庫金額を販売可能数量に戻すために新しい予約が入力されます。 予約は事実上互いにキャンセルし、どのソースからも数量を差し引かない。 その他の顧客は、関連する在庫および販売チャネルを通じて、積極的に製品数量を購入できます。

### 払い戻し注文

顧客が返金を要求した場合は、製品の一部または全額のクレジットメモを発行します。 返却された製品を受け取ったら、クレジット・メモを入力して資金を提供し、製品の金額を更新します。 「在庫に戻る」オプションを選択する場合、 [!DNL Commerce] は、注文と予約報酬を出荷した製品とソースに数量を追加して、関連する在庫の販売可能数量を更新します。

## 注文タイプ

単純な注文は、買い物かごから始まり、支払いを続行し、満足した配信で終わります。 この命令では [!DNL Inventory Management] は、買い物かごおよびチェックアウトでの可用性（または販売可能数量）に対する予約を容易に処理し、出荷時に手持在庫から差し引きます。

![単純な注文のプロセス](assets/diagram-simple-order-flow.png){width="600" zoomable="yes"}

より複雑なオーダーには、一部のキャンセル、部分出荷および返金が含まれる場合があります。 この場合、予約は、在庫に影響を与え、キャンセルや払い戻しの数量を追加し、発注時や出荷時に数量を減らします。

![複雑な注文のプロセス](assets/diagram-complicated-order-flow.png){width="600" zoomable="yes"}

在庫状況の予約と在庫の変更は、注文ステータスに基づいて行われます。

## ステータスと予約

次の表は、予約変更が入力されたオーダーとクレジット・メモのステータスの詳細を示しています。 [!DNL Commerce] をクリックして、在庫を管理します。

| 注文ステータス | 説明 | 販売可能数量の予約 |
|--|--|--|
| [!UICONTROL Open] | 新規および最近送信済み、処理なし | 注文が在庫に送信されると、予約が保存されます。 |
| [!UICONTROL Canceled] | 支払い前に一部または全部でキャンセル済み | 予約報酬が入力され、一部または全数量が在庫販売可能数量に戻されます。 |
| [!UICONTROL On Hold] | 未処理または請求済みの支払と出荷 | 予約は残っています。 |
| [!UICONTROL Suspected Fraud] | 不正により処理されませんでした | 承認またはレビュー中の場合、予約はそのまま残ります。<br/>拒否された場合、マーチャントが承認またはキャンセルを決定するまで、予約は維持されます。<br/>取り消された場合は、予約報酬が入力され、完全な数量が在庫販売可能数量に戻されます。 |
| [!UICONTROL Pending] | 支払いを待機中 | 予約はそのまま残る。 |
| [!UICONTROL Processing] | 支払い処理、未受信 | 予約はそのまま残る。 |
| [!UICONTROL Pending Payment] | 支払いを受け取っていません | 予約はそのまま残る。 |
| [!UICONTROL Payment Review] | 処理および完了の確認中の支払い | 予約はそのまま残る。 |
| [!UICONTROL Complete] | 全額で支払済みおよび出荷済み | 引当金額は、一部または全額の請求時に、選択したソースの製品数量から引き落とされます。 予約報酬が入力され、合計販売可能数量が更新されます。 |
| [!UICONTROL Closed] | 返金またはアーカイブ済み | アーカイブした場合、数量は変更されません。 一部または全額で払い戻された場合は、予約報酬が入力され、ソースごとの製品数量と在庫ごとの売上可能数量を加算するように変換されます。 |

| クレジットメモのステータス | 説明 | 販売可能数量の予約 |
|--|--|--|
| [!UICONTROL Open] | 払い戻しは期限で、未完了です | 予約に変更はありません。 |
| [!UICONTROL Refunded] | 完了、返金済み資金 | 一部または全額で払い戻しが行われた場合は、予約報酬が入力され、ソースごとの製品数量と在庫ごとの売上可能数量を加算するように変換されます。 |

## 複雑な順序の例

ブレイク・サンダースは、家族の休暇と楽しみのために自転車や服を注文します。 アメリカ、カナダ、ヨーロッパの在庫やソースを持つ Biking Adventures ストアでの素晴らしい販売を見ています。

彼らは小さな子供用に 2 台の大きなパークバイクを買います 10 代の子供用に BMX バイクを買います自分用に素敵なマウンテンバイクを買いますそして配偶者用に現代のドイツのクロスカントリーバイクを買います 可愛いシャツを売っていたので、家族全員で買ってくれました。 以下の休暇中の購入のリスト、対応する SKU、在庫販売可能数量に対して入力された予約を参照してください。

![複雑な順序](assets/diagram-order-complex.png){width="600" zoomable="yes"}

彼らは、彼らが見つけたものを家族に示すが、いくつかの変更を行う。 支払いが完了する前に、彼らは 33-BikeFun SKU の 2 つをキャンセルします（子供は彼らを好きではなかった）。 これは支払い保留中のための部分的なキャンセルなので、クレジットメモは不要です。 更新するには、 [!DNL Commerce] カナダ向けの販売可能数量の在庫を増やします。 注文は支払われ、すべての製品は、休暇に間に合うように到着して出荷されます。 [!DNL Commerce] 出荷済製品の出荷倉庫の販売可能数量とソース数量を更新します。

しかし、そのシャツは配偶者に合わなかった。 ブレイクは払い戻しを要求し、彼のシャツを送り返す。 クレジットメモの作成は、カナダのストックと配送倉庫に戻る 54-BikeLife シャツを 1 つ追加します。

- **発送済みの製品**  — 製品を購入し、出荷した場合 [!DNL Commerce] 在庫を更新します。 引当報酬は、出荷元から手持在庫数量控除に変換されます。 在庫の利用可能な販売可能数量の更新。

- **キャンセルされた製品**  — 在庫をキャンセルすると、 [!DNL Commerce] その商品の予約を削除します。 2 枚のシャツの部分的なキャンセルに対する売上可能数量を加算するために、予約報酬が在庫レベルに入力されます。 これは、ソースレベルの在庫数量には影響しません。

- **クレジットメモ/払い戻し済みの製品**  — 返品する在庫は、数量に戻す必要があります。 クレジットメモを発行する際に、在庫に戻ることを選択できます。 [!DNL Commerce] 製品の出荷元に在庫数量を追加します。 残りの予約を消去するには、予約報酬が入力されます。 販売可能数量は、更新された数量に対して再計算されます。
