# [!DNL Page Builder]のリリースノート

これらのリリースノートには、[!DNL Page Builder]のリリースが記載されており、次のものが含まれます。

![新機能](../assets/new.svg)

![修正された問題](../assets/fix.svg)修正と改善

![既知の問題](../assets/bug.svg)既知の問題

>[!IMPORTANT]
>
>2.4.3 リリース以降、[!DNL Page Builder]はMagento Open Sourceのバンドル拡張機能として利用できるようになりました。 これは現在、Adobe CommerceとMagento Open Sourceの両方のデフォルトのコンテンツ編集ツールであり、WYSIWG エディターを任意のサードパーティモジュールに置き換えることができます。

## 1.7.2 for Commerce 2.4.5

![新規](../assets/new.svg) **新しいラッパーエンティティ - [!DNL Columns] コンテナが列レイアウト用に導入されました** – 以前は、ユーザーが列を追加できるのは、[!DNL Tab]や[!DNL Row]など、別のコンテナ/ラッパーコンテンツタイプ内のみでした。 これで、[!DNL Column]には独自のラッパーがあり、ステージ上で直接追加できます。 また、[!DNL Columns] コンテナには、ユーザーがこのラッパーエンティティの設定を指定できるsettings エレメントがあります。<!--- PB-500-->

![新規](../assets/new.svg) **列グループを複製、非表示、削除する新しいメニューオプション** – これらの新しいオプションを使用すると、ユーザーはコンテンツタイプと同様に、[!DNL Columns] グループを複製、非表示、削除できます。<!--- PB-507-->

![新規](../assets/new.svg) <!-- Issue 594 -->**列グループに新しい複数行の列サポートが追加されました** – この追加により、ユーザーは1つの[!DNL Columns] グループ内の複数行の列を操作して、列レイアウトをより柔軟にすることができます。
<!--- PB-108-->

新しい[!DNL Columns] グループの使用について詳しくは、[ レイアウト – 列](./column.md)を参照してください。

## Adobe Commerce 2.4.2の場合は1.6.0

![新規](../assets/new.svg) <!-- Issue 558 -->**新規[!DNL Page Builder] スタイル設定** - [!DNL Page Builder]は、コンテンツのスタイル設定を大幅に改善しました。 この変更により、既存のコンテンツタイプスタイルを上書きする簡単なCSS セレクターを簡単に作成できるようになりました。 これらの変更により、すべてのコンテンツタイプに対して応答性の高い[!DNL Page Builder] テーマを作成できます。

![新規](../assets/new.svg) <!-- Issue 429 -->**行コンテナがオプションになりました** – 行コンテナを必要とせずに[!DNL Page Builder]でコンテンツを作成できるようになりました。 行に加えて、HTML、ブロック、ダイナミックブロック、列、タブのコンテンツタイプをステージに直接ドラッグ&amp;ドロップできるようになりました。

![新規](../assets/new.svg) <!-- Issue 636 -->**新しいレスポンシブビューポートスイッチャー** - ステージの上にある新しい管理者ビューポートスイッチャーボタンを使用して、異なるデバイス幅（ブレークポイント）で[!DNL Page Builder] コンテンツをプレビューできるようになりました。 ビューポートスイッチャーは、ストアフロントに表示されるときのコンテンツのスタイル設定を、様々なデバイスを使用してシミュレートします。

![新規](../assets/new.svg) <!-- Issue 637 -->**新しいブレークポイント固有のフォームフィールド** - コンテンツタイプフィールドの値を特定のビューポートブレークポイントに設定できるようになりました。 フィールドの横にあるアイコンのインジケーターは、コンテンツタイプフィールドの値が適用されるビューポートを示します。

![新規](../assets/new.svg) <!-- Issue 559 -->**事前に定義されたフォームフィールドのエントリを削除** – 余白、パディング、およびボーダー関連のプロパティ（境界線、境界線、幅、および境界線の半径）の詳細なフォーム設定が`default`に設定され、モジュールのCSSで値を定義できるようになりました。

![新規](../assets/new.svg) <!-- Issue 635, 557,  -->**ステージの間隔を追加** – 行と列は、ステージ上の含まれるコンテンツタイプの周囲に追加の間隔を付与するようになりましたが、ストアフロントでは許可されません。 追加のステージ間隔を使用すると、コンテナと含まれるコンテンツタイプの両方のマウスオーバーのオプションメニューに簡単にアクセスできます。

![修正済みの問題](../assets/fix.svg) <!-- MC-37348 -->**レビューへの自動スクロールを修正** - ストアフロントの製品レビューへの自動スクロールが修正されました。

![修正された問題](../assets/fix.svg) <!-- MC-36227 -->**修正された切り抜きコンテンツ** – 長いカテゴリと製品説明が切り抜かれなくなりました。

![修正済みの問題](../assets/fix.svg) <!-- MC-35402 -->**修正済みの全幅、全裁ち落としカテゴリのコンテンツ** - カテゴリのコンテンツを全幅または全裁ち落としレイアウトで表示できるようになりました。

![修正された問題](../assets/fix.svg) <!-- MC-37989 -->**セキュリティの強化**

## Adobe Commerce 2.3.5の場合は1.3.0

![新規](../assets/new.svg) **テンプレート** - [!DNL Page Builder]に、既存のコンテンツから作成でき、新しいコンテンツ領域に適用できるテンプレートが追加されました。 [!DNL Page Builder] テンプレートは、既存のページ、ブロック、動的ブロック、製品属性、カテゴリ説明のコンテンツとレイアウトの両方を保存します。 例えば、既存の[!DNL Page Builder] CMS ページをテンプレートとして保存し、そのテンプレートを（すべてのコンテンツとレイアウトを含む）適用して、サイト用のCMS ページをすばやく作成できます。 この新機能については、[ テンプレート ](templates.md)を参照してください。

![新規](../assets/new.svg) **行、バナー、スライダーのビデオ背景** - [!DNL Page Builder]行、バナー、スライダーのビデオを背景に使用できるようになりました。 これらの新機能については、[行](row.md)、[ バナー](banner.md)、[ スライダー](slider.md)を参照してください。

![新規](../assets/new.svg) **動画サポートの追加と機能強化** - [!DNL Page Builder]では、より幅広い様々な動画形式をサポートするようになりました。 YouTubeおよびVimeo ビデオに加えて、ビデオコンテンツの種類とビデオの背景で、`.mp4`などのビデオフォーマットへのビデオ URL リンクやその他のブラウザーがサポートするフォーマットがサポートされるようになりました。 ビデオコンテンツタイプには、自動再生機能も追加されています。 これらの新機能については、[ ビデオコンテンツタイプ ](video.md)を参照してください。

![新規](../assets/new.svg) **全高の行、バナーおよびスライダー** - [!DNL Page Builder]行、バナーおよびスライダーは、任意のCSS単位（px、%、vh、em）の数値または単位間の計算（100vh～237px）を使用して、ページの全高に高さを設定できるようになりました。 これらの新機能については、[行](row.md)、[ バナー](banner.md)、[ スライダー](slider.md)を参照してください。

![新規](../assets/new.svg) **コンテンツ型アップグレードライブラリ** – 開発者は、以前のバージョンとの下位互換性のない問題を導入することなく、[!DNL Page Builder] コンテンツ型のバージョンを作成できるようになりました。 このリリースの前は、コンテンツタイプ設定に大きな変更を加えると、以前に保存した[!DNL Page Builder]個のコンテンツタイプで表示とデータ損失の問題が発生していました。 新しいアップグレードライブラリは、これらの問題を解消します。 ライブラリは、データベースに保存されたコンテンツタイプの以前のバージョンをアップグレードし、新しいバージョンの設定変更と一致するように設計されています。 [!DNL Page Builder]は、新しいリリースの必要に応じて、ネイティブコンテンツタイプでアップグレードライブラリを実行します。 この変更により、ビルトイン [!DNL Page Builder]のコンテンツタイプが、新しいリリースでコンテンツタイプに加えられたすべての変更に合わせて常にアップグレードされるようになります。
