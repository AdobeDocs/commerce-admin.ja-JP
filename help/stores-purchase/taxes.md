# 税金

ロケールの要件に応じて税金を計算するようにストアを設定します。 製品と顧客グループに対して[税区分](tax-class.md)を設定し、製品と顧客の区分、税区分、および税率を組み合わせた[税区分](tax-rules.md)を作成できます。 Commerceでは、固定商品税、複合税、国際的な価格表示の設定も提供しています。 [付加価値税](vat.md)を徴収する必要がある場合は、ストアを設定して、検証を行って適切な金額を自動的に計算できます。

>[!NOTE]
>
>Adobe CommerceおよびMagento Open Source リリース 2.4.0 ～ 2.4.3には、Vertex Cloudとの統合に使用するVertex ベンダーが開発した拡張機能が含まれており、税金管理とアドレスのクレンジングを提供します。 2.4.4 リリース以降、この拡張機能はコアリリースにバンドルされなくなり、Commerce Marketplaceから、またはベンダーから直接インストールして更新する必要があります。 拡張機能とドキュメントについて詳しくは、[Vertex](https://marketplace.magento.com/partner/vertex_inc)にお問い合わせください。<br><br>
>
>バンドル拡張機能を有効にして設定している場合は、2.4.4 アップグレードプロセスの一環としてcomposer.json ファイルを更新し、拡張機能の更新を今後も管理する必要があります。 _アップグレードガイド_&#x200B;の「[&#x200B; モジュールのアップグレード &#x200B;](https://experienceleague.adobe.com/docs/commerce-operations/upgrade-guide/modules/upgrade.html?lang=ja)」を参照してください。

## クイックリファレンス

一部の税金設定では、税金の計算方法や顧客への表示方法を決定するオプションを選択できます。 詳しくは、[国際課税ガイドライン &#x200B;](international-tax-guidelines.md)を参照してください。

税金計算設定を設定する際は、次の表を参照してください。

### 税計算メソッド

税額計算オプションには、[!UICONTROL Unit Price]、[!UICONTROL Row Total]および[!UICONTROL Total]が含まれます。 次の表は、異なる設定に対する丸め（2桁まで）の処理方法を示しています。

| 設定 | 計算と表示 |
|--- |--- |
| [!UICONTROL Unit Price] | Commerceでは、各品目の税金が計算され、税込み価格が表示されます。 税合計を計算するには、各品目の税金を丸めて、それらを合計します。 |
| [!UICONTROL Row Total] | Commerceでは、各行の税金が計算されます。 税合計を計算するには、各品目の税金を丸めて、それらを合計します。 |
| [!UICONTROL Total] | Commerceは、各品目の税金を計算し、それらの税額を追加して、注文の合計非端数税額を計算します。 次に、指定した丸めモードを合計税に適用して、注文の合計税を決定します。 |

{style="table-layout:auto"}

### カタログ価格（税込み/税別）

使用可能な表示フィールドは、計算方法と、カタログ価格に税金が含まれているか除外されているかによって異なります。 通常の計算では、表示フィールドの精度は10進法で2桁です。 価格設定の一部の組み合わせでは、税込み価格と税抜き価格の両方が表示されます。 両方が同じ行項目に表示される場合、お客様にとっては混乱を招く可能性があり、トリガーは[warning](taxes.md#warning-messages)になります。

| 設定 | 計算と表示 |
|--- |--- |
| [!UICONTROL Excluding Tax] | この設定を使用すると、入力時に基準品目価格が使用され、税計算方法が適用されます。 |
| [!UICONTROL IncludingTax] | この設定を使用すると、税抜きの基準品目価格が最初に計算されます。 この値が基準価格として使用され、税計算方法が適用されます。 |

{style="table-layout:auto"}

>[!IMPORTANT]
>
>EU加盟店や他のVAT加盟店の以前のバージョンから、税金を含む価格を表示し、複数のストアビューを持つ複数の国で運営されているバージョンが変更されています。 2桁を超える精度で価格を読み込む場合、Commerceでは、購入者に一貫性のある価格が表示されるように、すべての価格が自動的に2桁に丸められます。

### 送料は税込みまたは税抜き

| 設定 | 表示 | 計算 |
|--- |--- |--- |
| [!UICONTROL Excluding Tax] | 税抜きのようです。 | 通常の計算： 配送はカートの合計に追加され、通常は別の商品として表示されます。 |
| [!UICONTROL Including Tax] | 税金を含めることも、税金を別に表示することもできます。 | 配送は、同じ計算を使用して、税金を含むカートの別の商品として扱われます。 |

{style="table-layout:auto"}

### 行項目としての税額

2つの異なる税額を、カナダの店舗のGSTやPSTなどの個別の品目として表示するには、関連する税金規則に対して異なる優先度を設定する必要があります。 しかし、以前の税計算では、異なる優先順位の税金が自動的に配合されます。 税額を誤って配合することなく個別の税額を正しく表示するには、異なる優先順位を設定し、「_小計のみ計算_」チェックボックスも選択します。 この設定では、別の項目として表示される税額が正しく計算されます。

## 警告メッセージ

税金関連の選択肢の中には、お客様に混乱を招き、警告をトリガーするものもあります。 これらの条件は、税額計算方法が`Row`または`Total`に設定され、お客様に税額控除と税額控除の両方の価格が提示された場合に発生する可能性があります。 また、カート内にアイテムごとに税金がある場合にも発生する可能性があります。 税額計算は丸められるので、カートに表示される金額は、顧客が支払うことを期待する金額とは異なる場合があります。

税計算が問題のある設定に基づいている場合は、次の警告が表示されます。

![感嘆符](../assets/icon-warning.png) **警告**。`Tax discount configuration might result in different discounts than a customer might expect for store(s); Europe Website (French), Europe Website (German). Please see source for more details.`

![感嘆符](../assets/icon-warning.png) **警告**。`Tax configuration can result in rounding errors for store(s): Europe Websites (French), Europe Websites (German).`

## デジタル商品の供給場所（EU）

欧州連合（EU）の加盟店は、四半期ごとに各加盟国に販売したデジタル商品を報告する必要があります。 デジタル商品には、顧客の輸送先住所に基づいて課税されます。 この法律では、小売業者に対して、物理的な商品ではなく、デジタル商品に関連する税額を特定し、報告書を作成することを義務付けています。

加盟店は、EU加盟国が四半期ごとに販売したすべてのデジタル商品を、期間中に収集した税金の支払い期限とともに中央税務署に報告する必要があります。

まだ基準値に達していない商人（年間事業額の50,000/100,000万ユーロ）は、VAT番号を登録したEU加盟国に販売された物理的な商品を引き続き報告する必要があります。

デジタル商品に対して支払われた税金の監査を受けるマーチャントは、顧客の居住地を確立するために、2つの支援情報を提供する必要があります。

- 顧客の配送先住所と支払い取引が成功した記録を使用して、顧客の居住地を確立することができます。 （支払いは、配送先住所が支払いプロバイダーの情報と一致する場合にのみ受け付けられます）。
- 情報は、Commerce データベーステーブルのデータストアから直接キャプチャすることもできます。

_&#x200B;**デジタル物品税情報を収集するには：**&#x200B;_

1. すべてのEU加盟国の税率を読み込みます。

1. デジタル商品商品商品商品税区分を作成します。

1. すべてのデジタル製品をデジタル製品税区分に割り当てます。

1. 物理的な商品に対して、物理的な商品税区分を使用して[税規則](tax-rules.md)を作成し、それらを適切な税率に関連付けます。

1. デジタル商品に対する製品税クラスを使用して、デジタル商品に対する税ルールを作成し、EU加盟国の適切な税率に関連付けます。

1. 適切な期間の税金レポートを実行し、必要なデジタル商品情報を収集します。

1. デジタル製品税区分の税率に関連する税額をエクスポートします。
